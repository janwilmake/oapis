OAPIS

Forget SDKs. Use APIs directly with instant TypeScript and JavaScript generation.

View on GitHub

Why OAPIS?

I've always been frustrated with the typical API integration workflow. Every API requires its own SDK, often bringing along hundreds of files and bloating your project. Take OpenAI's SDK for example - it's massive, when you might only need a single endpoint.

With OAPIS, you can:

  • Import single-file TypeScript or JavaScript functions for specific API operations
  • Use with Deno without any SDK installation
  • Generate man-page style documentation for any OpenAPI endpoint
  • Extract specific operations for targeted use in AI tools like ChatGPT
Example Usage 
// Get TypeScript for Stripe's create customer endpoint
const response = await fetch('https://oapis.org/ts/stripe.com/v1/customers');
const tsCode = await response.text();

// Generate man-page style docs for GitHub's repo API
const docs = await fetch('https://oapis.org/summary/api.github.com/repos/{owner}/{repo}');
